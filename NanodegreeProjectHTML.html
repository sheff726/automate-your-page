KEY CONCEPTS

LESSON ONE: BASICS OF THE WEB AND HTML

What is a web page?

The internet is made up of pages of HTML (Hyper Text Markup Language) that are stored on servers and transmitted through HTTP to individual computers.

How does Coding work?

Coding is writing text in a language that computers can understand. The primary language for this is HTML. HTML is made up of text content (what you see), markup (what it looks like), references to other documents (images, videos), and links to other pages.

Computers Are Stupid!

Computers are stupid, in that they only understand the language they know and if there is even a small error, they cannot understand what they are being asked to do. This makes learning how to make code exactly precise is required. Computers can do anything we want, IF we learn how to communicate with them!

Programmers can't remember everything!

There are too many details for anyone to remember. Learn the main concepts and structures, anything else (individual tags,etc.) can be looked up online through lots of great resources. (such as the Mozilla Developer Network or doing a Google search of what you are looking for)

Basic HTML Vocabulary
    Tag: commands inside brackets; for example, p for making a paragraph
    Element: everything written within a set of opening and closing tags; format is: opening tag, content, closing tag
    Attribute: a property of an HTML element; for example, href
Inline vs. Block Elements (Span and Div)

Huge concept to remember! Div is how you make BOXES around content on your page and really helps to organize the layout of the page. Also, it helps later with being able to style individual elements of your page. Span is the inline element and helps to style individual words, items, etc. within your boxes. For example, you can use span to make a word bold (and be more specific with how dark you want the boldness to be), instead of using the b tag. Understanding Span and Div really helps in visualizing how a webpage works!

LESSON TWO: CREATING A STRUCTURED DOCUMENT WITH HTML

Tree-like structure of HTML

When reading HTML as text, it has a tree-like structure and has boxes, nested within boxes, nested within boxes. This tree-like structure is referred to as DOM (Document Object Model). This contains HTML, which controls the structure of the page, and CSS, which controls the style of the page.

Boxes!

A webpage can be thought of as a collection of boxes, which have boxes within boxes, that can be moved around to restructure the page. Boxes are built on the page from large boxes to small. In the actual HTML text, these boxes within boxes can be seen by HTML that is indented. This is where Div comes into play again since it is how you make your boxes.

Text Editors for Programming

Using special text editors, such as Sublime Text or Codepen, help to make things easier on the programmer. For example, the program will guess what tag you are trying to enter or automatically generate a closing tag when you enter an opening tag! Also, they allow you to make changes and see results to decide if you want to keep your change or stay with what you had.

LESSON THREE: ADDING CSS STYLE TO HTML STRUCTURE

What is CSS?

CSS (Cascading Style Sheets) allows us to use code to avoid repetition. Also, allows us to separate our html and css into different files to keep our html neater, as well as allowing us to reuse our CSS in other projects. A relatively small amount of CSS can style even a large HTML document!

Inheritance in CSS

CSS properties are applied not only to a specified element, but to it's descendents, as well. This allows developers to not have to repeat code over and over! Also enhances faster loading web pages and saves on bandwidth used and development costs.

How Do I Include CSS Styling In My Web Page?

1. Write CSS in the head of your HTML. (This is good for small projects)

At the top of your HTML document, add a head element. Inside the head element, add a style element. Write your CSS in the style element and then put the HTML for the rest of your page below.

2. Link your HTML to a separate CSS file. This adds another step, but allows for easier organization of larger projects.

First, write all of your structural HTML in one file. Then, in a separate file, put all of your CSS code. Add a head element to the top of your HTML. Add a link tag inside of the head element. Then add the appropriate attributes to your link. Make sure to have your HTML file and CSS file contained within the same folder!

3. Write your style inline with your HTML. This is generally a bad idea which leads to a lot of repeated code. Also, increases the likelihood of mistakes since you are writing more code.

Positioning Boxes

To position boxes, create a flex-box layout. To do this, apply display:flex to the parent element. HTML elements are boxes and each box has four components. Because there are so many components to each box, getting the size of a single box right can be difficult. There are two techniques to help deal with sizing issues: one, set sizes in terms of percents, instead of pixels and two, set the box-sizing attribute to border-box for every element. (ex. box-sizing: border-box) This Clip does a good job of showing how to position and line up boxes next to each other.

LESSON TWO: VARIABLES AND STRINGS

What is a Variable?

Variables give programmers a way to gives names to numbers, in Python. This allows the programmer to only have to enter the assigned variable name instead of having to repeat a huge number or expression multiple times. Another great way to cut down on repetition!

What does it mean to Assign a value to your Variable?

When you assign a value to a variable, you can use that variable in expressions instead of having to repeat that value again. Also, this makes it easier for someone else to understand what you are doing and for you to understand if you want to make changes or have to debug later on. Additionally, variables can vary. That is, the value of a variable can be changed and carry that new value forward in the program.

What is the difference between what = means in math versus in programming?

In math, the = sign means "equal to", such as 2 + 4 = 6. In variables, the = sign means that it assigns a value to a variable. Another way to think of it is the = is like an arrow, pointing to the left, to assign whatever the expression answer is to the variable.

Why is the right side of the expression evaluated first?

The right side of the expression is evaluated first to determine what value to assign to the variable. Like above, the = sign functions like an arrow to the left.

Strings

A string is any sequence of characters between quotes in Python. The only rule for strings is that they have to open and close with either a single or double quote (same on each side). Also, using a + between two strings means a concatenation of two strings (pasting them together).

Indexing Strings

Indexing a string is writing code to identify parts of the string, [3:] for example. This can also be thought of as extracting subsequences. We can also select subsequences by entering string=[expression start:expression stop]. Note, the expression stop will include everything up to it's position, but not it's position. To get all characters from a position, leave one side empty.

Finding Strings in Strings

Use variable.find('string') to find the first position where the target string occurs. To find multiple occurences of the string, use variable.find('string', number). This adds a second Parameter to find. Very powerful extraction tool, once we learn how to use it!

LESSON THREE: INPUT->FUNCTION->OUTPUT

Functions

A function takes an input(s) and uses it to produce an output. For example, a function called Square might take a number as an Input and produce that number squared as an Output.

What is the difference between Making a function and Using a function?

A Function is made by starting a line of code with the keyword def and then giving a function name, followed by the function parameters, in parentheses. Then we write code that specifies what to do with the input parameters.

def = square(x): *Remember the colon!!!!
answer = x + x
return answer

To use the function, we write the name of the function followed by the value(s) for the parameters. (Keyword print does this)

print square(4)
>>>>16

What happens if a Function doesn't have a Return statement?

The return keyword tells Python what the function should output. Without this, the function, well, doesn't function and you get a result of >>>>None (essentially an error statement).

How do Functions help programmers avoid repetition?

Functions are tools that programmers can create and reuse forever. Once a function is defined, you don't have to define it again! This can be used to do simple calculations or to help build structural HTML in a page like this one!

LESSON FOUR: IF AND WHILE STATEMENTS

If Statements

If statements control which code gets executed when and helps the program to make decisions! An if statement consists of if followed by a test expression on the first line. On the next line, indented, is the block, which only executes if the test expression is True. If the block is False, the Block will not run and the code moves on. Decisions start to be made by adding else and another block after this. If there are multiple tests, use multiple ifs.

Also, you can include an Else, which provides an additional block that will execute if the original block returns False.

Use of Operators and Boolean Values

Operators (>, <, ==, etc) are a big assist in creating comparisons to make decisions. These comparisons result in a Boolean Value, which means they evaluate to either True or False.

Other operators to remember:

!= means not equal to

== means equal to (remember that = already means assignment in Python)

The Or Operator

Expression or Expression

If the first expression evaluates to True, the value is True and second expression is not evaluated. If the first expression is False, then the value of the second expression is returned.

Loops

Loops use the while statement with a block afterwards and can run an infinite amount of times, as long as the test expression is true. An infinite loop is when a loop never stops. This is usually a result of a programming error, what we commonly know as a "bug". We can use a break statement to stop a loop statement, even when it is still true. Once the break test evaluates to true, it will move on in the code.

LESSON FIVE: DEBUGGING STRATEGIES

Examine Error Messages When Programs Crash

In Python, the error messgage (Traceback) show the line where the first error occurs and gives a description of the problem. If an error message doesn't help, use Google to help try to understand the error message.

Work From Example Code

If you build your own code using other code as an example, make sure to check your work if the code doesn't work as you expected. Make sure that the example code works, then, if it does, work through yours to see where you have made an error.

Make Sure Examples Work

When using example codes, make sure to check that it actually does what you want or expect it to. If there is a bug, make small changes and check them to locate the bug.

Check (print) Intermediate Results

When a bug doesn't crash your code, but instead gives you an incorrect result, it is best to break it down into smaller pieces to see what isn't working as you intended. One way is to add print statements after variables to see if they are working as intended. (Remember to remove the Print statements!!)

Keep and Compare Old Versions

Sometimes, when code isn't working, you may be tempted to throw it away and start over. That's ok, but make sure to keep a copy of your current version. You may actually be closer to what you want than you think you are. Also, if you later go to forums or others for help, being able to show what you have tried can help others to help you!

LESSON SIX: STRUCTURED DATA: LISTS & FOR LOOPS

Structured Data

Structured Data is refers to Strings and Lists. While strings can only be comprised of a sequence of characters, Lists can be a sequence of anything. Lists can have numbers, strings, other lists, etc. inside of them, they also do not have to be the same throughout the list. You can access elements of the list just like you do strings, by placing the position number you want inside of brackets.

Mutation

In a list, a value within the list can be changed and reassigned. Elements can be changed to a new value, with an assignment statement, which also changes the value of the list.

You can also mutate variables (intentionally and unintentionally). If you introduce a new variable and it is assigned the same value as another variable, when the value of one of them is changed, so is the other.

Aliasing

Aliasing refers to there being two different ways to refer to the same program. If two variables have the same value, if you modify the value for one, it is modified for the other as well. BUT, if you assign a new value to one variable, the other variable remains the same. Also, you can use an assignment statement to replace a value in the list. (can alter one part of the list in an assignment statement in a procedure).

List Operations

append : Adds a new element to the end of a list
+ : Concatenates the list and produces a new list
len : Gives the length of a list (works on strings also)
Difference between append and +: Append does NOT produce a new list, while the + operator adds the lists together and makes a new list.

For Loops

Sets up a loop to run through all elements in a list. Has a block, just like a while loop.
for *name* in *list*:
-----*block*
You can also use the Index function, which will return the first position where the value appears in the list. If there is no match, it will return an error.
*list*.index*value*
There are two other ways to search a list, In and Not In. These will return a Boolean Value, True or False.
*value* in *list*

LESSON SEVEN: HOW TO SOLVE PROBLEMS

Understanding the Problem

A problem is defined by possible inputs and the relationship between those inputs and desired outputs. Thinking about the problem like a human would solve it is a good place to start.

Guide to Solving Problems

1. What are the Inputs?
2. What are the Outputs?
3. Solve the Problem
4. Simple Mechanical Solution
5. Develop Incrementally and Test as you go

Notes on Guide to Solving Problems

When solving the problem, work through some examples to understand the relationship between the Inputs and the Outputs and consider Systematically how a human would solve this problem.

Be as simple as possible in your solutions. Making progress on a problem is writing small bits of code and testing them to know what they do.

Use lots of testing, pseudocode, and helper functions as tools to help you build your solution to the problem. Another tool to help is to use Assertion statements. These can be added to test for odd cases and to make sure code is functioning as intended. The assertion will return a False if it fails.
